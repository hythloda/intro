Other ways to join

There are several ways to become a validator in the Canton Network. If you want the fastest path to participation, you can use a white-label validator node operated by an approved Canton Node-as-a-Service provider.

This option gives you network access without needing to deploy or maintain your own infrastructure. You can review the full validator access options here: canton.foundation/validators

  • Join the network quickly with minimal setup.
  • Rely on expert operators who manage security, uptime, and compliance.
  • Maintain your own secure identity and permissions within the network.
  • Keep the flexibility to transition to operating your own validator in the future.

The Foundation also lists approved Node-as-a-Service providers and explains how to apply as a validator operator or request recognition as a service provider.

How Canton works

Canton is a network of networks. Instead of globally replicating all state to every node, data is distributed only to the validators and parties that need to see it. That privacy-by-design model is one of the biggest differences between Canton and public blockchains.

Validators

Your validator stores contract data and executes smart contract code for the parties it hosts.

  • Your validator only sees contracts where your parties are stakeholders.
  • Validators have state and are not meant to be interchangeable or ephemeral.
  • Each validator connects to one or more synchronizers.
Global Synchronizer

The Global Synchronizer is the coordination and ordering layer operated by Super Validators.

  • It time-orders cross-application transactions.
  • It forwards encrypted messages without understanding transaction contents.
  • It prevents conflicts while preserving privacy.

Compared with public blockchains, Canton uses selective data distribution, party-based identity, and validator-hosted private data rather than broad state replication across all nodes.

1. Sponsorship Requirement

  • To join the Canton Network as a validator, you must have a sponsor.
  • If you do not have one, you can request sponsorship from the Canton Foundation.

2. Initial Application

Start by completing the validator application form:

Apply to set up a validator node

3. Tokenomics Committee Approval

  • After submitting the application, you must be approved by the Tokenomics Committee.
  • This approval is required for both TestNet and MainNet.
  • If you only want DevNet access, you can skip Tokenomics Committee approval and submit the IP request directly here: DevNet IP Request.
  • The committee reviews applications weekly, every Wednesday.
  • Results are published here: tokenomics-announce
  • You cannot proceed to TestNet or MainNet without this approval.

4. MainNet Validator Allocation Process

  • A weekly validator allocation process is used to manage MainNet access.
  • Priority is given to GTM-ready projects.
  • GTM-ready projects include those with functional builds on DevNet or TestNet and readiness for immediate MainNet activation.
  • Capacity is up to 35 validators per week.
  • Allocations are processed every Wednesday.
  • A new Monday.com intake process is now used for MainNet allocation requests. Submit here: MainNet Allocation Request Form
  • Queue details and the current Monday board snapshot are available here: MainNet Allocation Queue
  • Announcements are published here: supervalidator-announce.
  • More details: Validator Allocations Jan 2026
  • Entries can be verified here: Validator Allocation Spreadsheet

5. After approval

  • After approval, continue to the post-approval onboarding page for DevNet and MainNet operational steps.
  • That page covers IP whitelisting, node setup, sponsor coordination, onboarding secrets, and verification scripts.
  • Use the official docs for the technical setup steps once you reach that stage.

Infrastructure and deployment expectations

  • After approval, provide a static egress IP address for whitelisting.
  • Prepare a container-based environment on a VM or Kubernetes cluster.
  • OIDC authentication is typically required, using a provider such as Auth0 or Keycloak.
  • Kubernetes is recommended for production deployments.
  • You can deploy in cloud or on-premise environments depending on your operating model.

The common path is to get stable on DevNet first, then request TestNet access, and finally move to MainNet when the validator and application flows are production-ready.

Summary Flow

  1. Secure a sponsor.
  2. Submit the validator application form.
  3. Wait for Tokenomics Committee approval through the Wednesday cycle.
  4. Submit the Monday MainNet allocation form if you are targeting MainNet access.
  5. Review the MainNet allocation queue and weekly timing.
  6. Continue to post-approval onboarding for whitelisting and deployment steps.
